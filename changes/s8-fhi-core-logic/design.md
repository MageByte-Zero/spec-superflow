# S8 · FHI 核心逻辑 — 设计文档 (V3.0)

## Context

### 当前状态

FHI 评测量表是若谷平台的核心评测工具。目前前端 6 个占位页面（6 维度 mock）与 V3.0 规格严重脱节。后端评分引擎公式与 V3.0 不一致，题库 JSON 仅 1 题桩。

### 约束

| 约束 | 说明 |
|------|------|
| 现有后端 API | `POST /api/v1/fhi/responses` 等已存在，响应需向后兼容扩展 |
| 前端框架 | uni-app Vue3 + TypeScript，微信小程序环境 |
| 数据源 | 题库 `shared/fhi/fhi-2.0.0.json` 前后端共享 |
| 评分引擎 | `backend/src/shared/fhi/score.ts` 纯函数 |
| Canvas | 微信小程序支持 `uni.createCanvasContext`，不支持 ECharts |

### 涉众

| 角色 | 关注点 |
|------|--------|
| 来访者 | 流畅答题，清晰的双图报告，可执行的改善建议 |
| 若谷顾问 | 通过环形图+雷达图快速识别家庭能量状态 |
| 开发者 | 计算公式准确，环形图/雷达图渲染正确 |

## Goals

1. **完全对齐 V3.0**：评分公式、双图系统、三级预警、12 型识别、8 节报告
2. **评分准确性**：`(原始分/原始满分)×权重` 公式无误，4 道反向题正确处理
3. **双图渲染**：环形图 3 环颜色/饱和度正确，雷达图四象限布局精确
4. **响应时间**：答题提交→结果展示 ≤ 3 秒

## Decisions

### Decision 1: 环形图 Canvas 实现

| 项目 | 内容 |
|------|------|
| **选择** | uni-app Canvas API 原生绘制环形图（3 层同心圆环） |
| **理由** | 微信小程序不支持 SVG；环形图结构固定（3 环），无需第三方库 |
| **颜色方案** | 外环(根基层) #4A90D9 蓝色、中环(能力层) #E8833A→#5CB85C 渐变、内环(目标层) #F5A623 金色 |
| **替代方案** | 第三方图表库 → 增加包体积且不必要 |

### Decision 2: 雷达图四象限布局

| 项目 | 内容 |
|------|------|
| **选择** | 8 轴 Canvas 雷达图，按四象限定位（0°/90°/180°/270°） |
| **布局** | 0°=关系层(关系亲密度/情绪环境)、90°=进化层(韧性成长力/系统规划力)、180°=共识层(价值稳定性/精神文化)、270°=基础层(结构稳定性/资源保障性) |
| **层级背景色** | 基础层 rgba(56,82,104,0.15)、关系层 rgba(124,155,176,0.15)、共识层 rgba(189,168,148,0.15)、进化层 rgba(162,184,165,0.15) |

### Decision 3: V3.0 评分公式

| 项目 | 内容 |
|------|------|
| **选择** | `维度得分 = (该维度原始分之和 / 该维度原始满分) × 维度权重` |
| **理由** | V3.0 官方指定；避免每题分配权重的复杂度；维度级计算更直观 |
| **影响** | JSON 中取消每题的 `weight` 字段；score.ts 逻辑从逐题改为逐维度 |

### Decision 4: 维度级预警

| 项目 | 内容 |
|------|------|
| **选择** | 检测维度级得分（关系亲密度<7.2、情绪环境<5.6），而非子维度级 |
| **理由** | V3.0 明确指定「维度分<40%」；与评分公式一致（维度级） |
| **影响** | 亲子信任和夫妻支持共享同一个阈值(关系亲密度<7.2) |

### Decision 5: 完整报告后端组装 vs 前端拼装

| 项目 | 内容 |
|------|------|
| **选择** | 后端返回结构化数据，前端按 8 节模板渲染 |
| **理由** | 评分引擎已产出所有需要数据；前端仅做展示不涉及计算；便于后期改版 |
| **影响** | API 响应结构需包含所有报告章节所需的原始数据 |

### 数据流 — 答题提交

```
wizard.vue → POST /api/v1/fhi/responses
  → routes/v1/fhi.ts
  → score.ts.scoreAnswers(form, answers):
     1. 逐维度计算原始分和（含4题反向）
     2. 维度得分 = (原始分/原始满分) × 权重
     3. 层级得分和比值
     4. 总分
     5. 12型识别
     6. 三级预警检测
     7. 环形图能量流
     8. 若谷心语
     9. 4层建议
  → DB 存储完整结果
  → 响应含所有报告章节数据
```

## Risks And Trade-Offs

| 风险 | 可能性 | 影响 | 缓解 |
|------|:------:|:----:|------|
| 环形图 Canvas 在小程序端渲染异常 | 低 | 中 | 使用 uni-app 封装的 Canvas API |
| 中环满分值 V3.0 改为 50（含4维度）与 V2.0 的 38 不同 | 中 | 高 | 以 V3.0 为准，文档和代码保持一致 |
| 维度级预警将亲子信任和夫妻支持合并检测 | 低 | 中 | 预警触发后需标注具体薄弱子维度名称 |
| 8 节报告导致结果页过长 | 低 | 低 | 使用折叠卡片可展开/收起 |
